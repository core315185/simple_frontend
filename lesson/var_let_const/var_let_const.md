# var, let, const の違い

JavaScriptには変数を宣言する3つのキーワードがあります。

---

## var

- **スコープ**: 関数スコープ（functionの中でのみ有効）
- **再宣言**: 同じ名前で何度でも宣言できる
- **再代入**: できる
- **巻き上げ（hoisting）**: 宣言が先頭に巻き上げられ、`undefined` として扱われる

```javascript
function example() {
  console.log(x); // undefined（エラーにならない）
  var x = 10;
  console.log(x); // 10
}

var x = 1;
var x = 2; // 再宣言できる（エラーにならない）
```

**問題点**: スコープが広いため、意図しない場所で変数が変わるバグが起きやすい。

---

## let

- **スコープ**: ブロックスコープ（`{}` の中でのみ有効）
- **再宣言**: 同じスコープ内では不可
- **再代入**: できる
- **巻き上げ**: されるが、初期化前にアクセスするとエラーになる（一時的デッドゾーン）

```javascript
let count = 0;
count = 1; // 再代入はOK

// let count = 2; // エラー：同じスコープで再宣言は不可

if (true) {
  let blockVar = "ブロック内";
  console.log(blockVar); // "ブロック内"
}
// console.log(blockVar); // エラー：ブロックの外では使えない
```

**用途**: 値が変わる変数（ループのカウンターなど）に使う。

---

## const

- **スコープ**: ブロックスコープ（`{}` の中でのみ有効）
- **再宣言**: 同じスコープ内では不可
- **再代入**: できない
- **巻き上げ**: されるが、初期化前にアクセスするとエラーになる

```javascript
const PI = 3.14;
// PI = 3.15; // エラー：再代入は不可

// オブジェクトや配列の中身は変更できる（参照先は変わらないため）
const user = { name: "Alice" };
user.name = "Bob"; // これはOK
// user = {}; // エラー：変数自体への再代入は不可
```

**用途**: 変わらない値（定数、オブジェクトの参照など）に使う。

---

## まとめ表

| 項目 | var | let | const |
|------|-----|-----|-------|
| スコープ | 関数スコープ | ブロックスコープ | ブロックスコープ |
| 再宣言 | できる | できない | できない |
| 再代入 | できる | できる | できない |
| 巻き上げ | undefined として扱われる | エラー（一時的デッドゾーン） | エラー（一時的デッドゾーン） |

---

## 現代のベストプラクティス

1. **基本は `const` を使う**
2. 値が変わる場合のみ **`let` を使う**
3. **`var` は使わない**（古いコードとの互換性のためだけに存在する）
